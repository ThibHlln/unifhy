  • Guide for Contributors
    • Preparation
      • Create your component by subclassing a generic framework component
      • Document your component by filling in your class docstring
      • Define your component by using component class attributes
      • Implement the initialise-run-finalise component class methods
    • Submission
      • Get your own copy of the framework repository
      • Create a branch on your copy
      • Commit your component to your branch in your local copy
      • Push your changes to your remote copy
      • Open a pull request to merge your component in the framework
